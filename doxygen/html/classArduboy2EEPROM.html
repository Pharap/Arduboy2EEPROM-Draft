Detailed Description

A class containing EEPROM-manipulating static functions.

Warning
The Arduboy has 1KiB of EEPROM, which spans the consecutive range of addresses from 0 to 1023 inclusive. Attempting to write to or read from any address beyond that range that range shall result in undefined behaviour.

Reads both an object and a hash code from EEPROM at the specified address, and determines if the hash of the object matches the stored hash code. This overload accepts a custom hash provider.

Complexity
The complexity of this function is equivalent to the complexity of the expression hash(object), where hash and object are the hash and object parameters of this function.
Parameters
[in]addressThe address of the hash code and object to be read.
[out]objectA reference to an object that shall receive the data read from EEPROM.
[in]hashA hash provider. Can be any type that supports an operator(), thus functions, function pointers, lambda expressions, and classes and structs with operator()s are all valid options.
Returns
  • true if the hash of the object matched the stored hash code.
  • false if the hash code did not match.
Precondition
  • (address <= 1023) — address must not exceed a value of 1023.
  • ((address + sizeof(hash_type) + sizeof(object)) <= 1024) — The value of the expression (address + sizeof(hash_type) + sizeof(object)) must not exceed a value of 1024.
  • Type must be a trivial type. — In particular, if it is a struct or class type, that type must not have any virtual functions and must not have any virtual base classes.
  • hash(object) must be a valid expression.
See also
